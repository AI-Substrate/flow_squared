## Executive Summary

SCIP (Source Code Intelligence Protocol) by Sourcegraph is a **drop-in replacement candidate** for Serena's cross-file relationship discovery. After installing and testing indexers for Python, TypeScript, Go, and C#/.NET, we confirmed that:

1. **Output format is universal** — all indexers produce identical protobuf structure
2. **Cross-file edges are trivially extractable** — match symbol definitions in file A with references in file B
3. **A single adapter can handle all languages** — only the scheme/manager prefix differs
4. **No runtime server needed** — SCIP runs offline, produces a static `.scip` file (unlike Serena's LSP pool)

### Symbol Format (UNIVERSAL across all indexers)

```
<scheme> <manager> <package> <version> <descriptor>
```

| Language | Scheme | Manager | Example Symbol |
|----------|--------|---------|---------------|
| Python | `scip-python` | `python` | `scip-python python pkg 0.1.0 \`module\`/Class#method().` |
| TypeScript | `scip-typescript` | `npm` | `scip-typescript npm . . \`file.ts\`/Class#method().` |
| Go | `scip-go` | `gomod` | `scip-go gomod mod hash \`path\`/Type#Field.` |
| C# | `scip-dotnet` | `nuget` | `scip-dotnet nuget . . Namespace/Class#Method().` |

### Descriptor Suffixes (STANDARD)

| Suffix | Meaning | Example |
|--------|---------|---------|
| `/` | Package/Module | `\`model\`/` |
| `#` | Type/Class/Enum | `TaskService#` |
| `.` | Field/Property/Enum value | `Priority#High.` |
| `().` | Method/Function | `AddTask().` |
| `(param)` | Parameter | `handle_request().(name)` |
| `:` | Module init | `\`handler\`/__init__:` |

### SymbolRoles (STANDARD bitmask)

| Value | Meaning | Seen In |
|-------|---------|---------|
| 0 | Reference (unspecified) | All |
| 1 | Definition | All |
| 8 | ReadAccess (import) | Python, TypeScript |

### Range Format (STANDARD)

- 3 elements: `[line, col_start, col_end]` (single-line occurrence)
- 4 elements: `[start_line, start_col, end_line, end_col]` (multi-line)
- Lines are 0-indexed

### Local Symbols

All indexers produce `local N` symbols for unnamed/anonymous constructs. These are file-scoped and can be ignored for cross-file analysis.

## How Cross-File Edges Work in SCIP

The algorithm is simple and **identical for all languages**:

```python
definitions = {}  # symbol_string -> file_where_defined
references  = {}  # symbol_string -> [files_where_referenced]

for document in scip_index.documents:
    for occurrence in document.occurrences:
        if occurrence.symbol_roles & 1:  # Definition bit
            definitions[occurrence.symbol] = document.relative_path
        else:
            references.setdefault(occurrence.symbol, []).append(document.relative_path)

# Cross-file edges = ref in file A, def in file B
for symbol, ref_files in references.items():
    if symbol in definitions:
        def_file = definitions[symbol]
        for ref_file in ref_files:
            if ref_file != def_file:
                yield (ref_file, def_file, symbol)  # source → target
```

## Comparison: SCIP vs Current Serena Approach

| Aspect | Serena (current) | SCIP (proposed) |
|--------|-----------------|-----------------|
| **Architecture** | Runtime LSP pool (15 instances) | Offline batch indexing |
| **Speed** | 29s for 3634 nodes (parallel) | <5s for 3-file fixtures |
| **Startup** | Spawn processes, wait for ready | Run CLI command, read file |
| **Languages** | Python, TypeScript (LSP-dependent) | 10+ languages, all consistent |
| **Port management** | Needs 15 consecutive ports | No ports needed |
| **Data format** | JSON responses per symbol | Single protobuf file |
| **Cross-file rels** | `find_referencing_symbols` per node | Match defs/refs in index |
| **Dependencies** | Serena binary + config | Per-language SCIP indexer |
| **Error handling** | Timeout per node, port conflicts | Single CLI exit code |
| **Incremental** | Per-unchanged-file cache | Re-index (fast enough?) |

### Key Advantages of SCIP

1. **Deterministic** — same input always produces same output
2. **No runtime state** — no port conflicts, no process management
3. **Language-agnostic adapter** — one parser for all languages
4. **Debuggable** — `scip print` shows human-readable output
5. **Community maintained** — Sourcegraph actively develops indexers

### Key Risks

1. **Per-language tool installation** — user must install correct indexer
2. **Project file required** — most languages need build files (not just raw dirs)
3. **Protobuf dependency** — need `protobuf` package to read `.scip` files
4. **Indexer freshness** — some indexers less maintained than others
